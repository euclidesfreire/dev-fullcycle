# Curriculum Map

Mapa inicial da formação. O conteúdo detalhado dos livros será planejado antes da escrita dos capítulos.

## Trilhas obrigatórias

1. Fundamentos de computação
2. Engenharia de software
3. Desenvolvimento full stack
4. Backend com Java Spring Boot
5. Backend com Node.js e TypeScript
6. Frontend com React e TypeScript
7. Bancos de dados
8. Sistemas distribuídos
9. DevOps
10. Cloud
11. Inteligência artificial aplicada
12. LLMs
13. RAG
14. Agentes de IA
15. Cybersegurança
16. Segurança em aplicações
17. Segurança em APIs
18. Segurança em IA
19. Projeto final integrado

## Regra de evolução

Cada capítulo deve declarar pré-requisitos, competências, dependências e conexão com o projeto final antes de ser aprovado.

### Projeto final do Livro 1

Ao longo dos capítulos, o leitor prepara um sistema mínimo, versionado, testável, seguro por design e observável. O objetivo não é competir com os projetos das trilhas Full Stack, IA ou Cybersegurança, mas criar a base mental e técnica para entendê-los com rigor.

## Livro 2 — Desenvolvimento Full Stack Moderno

Objetivo: capacitar o leitor a construir produtos Web completos, integrando frontend, backend, APIs, autenticação, persistência, testes, observabilidade e entrega contínua com visão profissional de produto.

Pré-requisitos: Livro 1 aprovado conceitualmente; Git; HTTP; programação básica; fundamentos de testes, segurança e arquitetura.

Competências:

- Modelar produtos Web end-to-end.
- Projetar contratos entre frontend e backend.
- Implementar autenticação, autorização e persistência.
- Testar fluxos críticos de produto.
- Operar uma aplicação full stack com observabilidade mínima.

### Partes

- Parte I — Fundamentos de produto Web: Arquitetura de aplicações Web modernas; Ambiente profissional de desenvolvimento full stack; Modelagem de domínio para produto digital; Contratos HTTP, JSON e OpenAPI.
- Parte II — Construção end-to-end: Backend mínimo para produto Web; Frontend mínimo com componentes e rotas; Persistência relacional aplicada; Autenticação, autorização e sessões.
- Parte III — Qualidade, operação e entrega: Testes full stack; Tratamento de erros e experiência do usuário; Observabilidade em aplicações Web; CI/CD para aplicações full stack.
- Parte IV — Produto integrador: Escalabilidade inicial e cache; Segurança full stack aplicada; Arquitetura evolutiva do monólito modular; Projeto prático full stack do Livro 2.

## Livro 3 — Backend Profissional com Java Spring Boot e Node.js

Objetivo: formar proficiência em backend profissional usando Java Spring Boot e Node.js com TypeScript, enfatizando APIs robustas, domínio, persistência, mensageria, testes, segurança e operação.

Pré-requisitos: Livros 1 e 2; domínio de HTTP, Git, programação, APIs, banco relacional básico e testes automatizados.

Competências:

- Projetar APIs profissionais.
- Escolher entre Java Spring Boot e Node.js por contexto.
- Aplicar camadas, domínio e casos de uso.
- Integrar persistência, filas e cache.
- Garantir qualidade, segurança e observabilidade no backend.

### Partes

- Parte I — Fundamentos profissionais de backend: Papel do backend em sistemas modernos; Design de APIs REST profissionais; Arquitetura em camadas, hexagonal e modular; Validação, erros e contratos de API.
- Parte II — Java Spring Boot: Spring Boot essencial para produção; Persistência com Spring Data JPA; Segurança com Spring Security; Testes em aplicações Spring Boot.
- Parte III — Node.js e TypeScript: Node.js profissional com TypeScript; APIs Node com validação e contratos; Persistência no ecossistema Node; Testes em backends Node.js.
- Parte IV — Integração e operação: Mensageria, eventos e jobs assíncronos; Cache, idempotência e consistência; Observabilidade e performance de APIs; Projeto prático backend do Livro 3.

## Livro 4 — Frontend Profissional com React e TypeScript

Objetivo: capacitar o leitor a construir frontends profissionais com React e TypeScript, cobrindo arquitetura de UI, estado, dados remotos, acessibilidade, testes, performance, segurança e design systems.

Pré-requisitos: Livros 1 e 2; JavaScript moderno; TypeScript básico; HTTP; componentes Web; noções de UX e testes.

Competências:

- Modelar interfaces orientadas a estado e domínio.
- Criar componentes reutilizáveis e acessíveis.
- Integrar APIs com cache e tratamento de erro.
- Testar experiência do usuário.
- Otimizar performance e segurança no navegador.

### Partes

- Parte I — Base profissional de React: React moderno e modelo mental; TypeScript aplicado a interfaces; Componentes, composição e props; Estado local e formulários complexos.
- Parte II — Dados, rotas e arquitetura: Roteamento e layouts de aplicação; Dados remotos, cache e sincronização; Arquitetura frontend por features; Design system e documentação de componentes.
- Parte III — Qualidade e experiência: Acessibilidade e usabilidade técnica; Testes de componentes e fluxos; Performance frontend; Segurança no frontend.
- Parte IV — Produto integrador: Autenticação e autorização na UI; Observabilidade frontend; Entrega e versionamento de frontend; Projeto prático frontend do Livro 4.
