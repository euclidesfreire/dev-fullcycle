# Quality Checklist

Um capítulo só pode ser marcado como completo quando todos os itens abaixo forem satisfeitos e auditados.

- [ ] Segue o template editorial obrigatório.
- [ ] Vai do básico ao especialista.
- [ ] Tem explicação conceitual profunda.
- [ ] Tem exemplo simples.
- [ ] Tem exemplo profissional.
- [ ] Tem aplicação em sistema real.
- [ ] Tem seção de segurança.
- [ ] Tem seção de performance.
- [ ] Tem seção de testes.
- [ ] Tem limitações e trade-offs.
- [ ] Tem erros comuns.
- [ ] Tem exercícios.
- [ ] Tem desafio prático.
- [ ] Tem conexão com o projeto final.
- [ ] Passa nos scripts de validação.
- [ ] Passa pela auditoria de profundidade.

# Checklist de profundidade temática dinâmica

Para cada capítulo, avaliar:

1. O capítulo foi analisado pelo `Topic Discovery Agent`?
2. Os temas principais foram identificados?
3. Os temas secundários foram identificados?
4. O papel de cada tema foi classificado?
5. O nível esperado foi calibrado?
6. Os perfis existentes foram aplicados?
7. Temas sem perfil foram registrados?
8. O conteúdo cumpre o nível esperado de cada tema?
9. O capítulo evita profundidade artificial?
10. O capítulo evita superficialidade em temas centrais?
11. O capítulo não exige deep-dive quando o tema é apenas mencionado?
12. O capítulo exige deep-dive quando o tema é central e especialista?
